An Identity sets contains a set of Identities that can be provisioned as accounts in target systems ("User" type Identity Set), or external users who don't require any accounts ("Contact" type Identity sets). You might choose to have multiple "User" sets, for example, to separate "Staff" from "Contractors" to make it easy to restrict the access of Contractors.
